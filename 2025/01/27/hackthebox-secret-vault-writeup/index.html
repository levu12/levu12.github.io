HackTheBox Web Challenge: Secret Vault Writeup

Welcome to my first detailed CTF writeup! Today I’m walking through “Secret Vault,” a medium-difficulty web challenge from HackTheBox that involves multiple exploitation techniques to capture the flag.

Challenge Overview

Platform: HackTheBox
Category: Web
Difficulty: Medium
Points: 300
Description: “A secret organization has developed a vault system to store sensitive documents. Can you break in and retrieve the flag?”

Initial Reconnaissance

Starting with basic reconnaissance of the target application:

The scan reveals:

  • Port 80: HTTP (Apache 2.4.41)
  • Port 22: SSH (OpenSSH 8.2p1)

Navigating to the web application, I find a login portal with the following features:

  • User authentication system
  • Document upload functionality
  • Admin panel (restricted access)

Lessons Learned

This challenge demonstrated several important security concepts:

  1. Input Validation: The application failed to properly sanitize user input, leading to SQL injection
  2. File Upload Security: Lack of file type restrictions enabled arbitrary code execution
  3. Defense in Depth: Multiple vulnerabilities chained together for full compromise

Mitigation Strategies

To prevent these vulnerabilities:

  • Use parameterized queries for database interactions
  • Implement strict file upload validation
  • Apply proper input sanitization
  • Regular security testing and code reviews

Tools Used

  • sqlmap: Automated SQL injection testing
  • Burp Suite: Web application testing proxy
  • Custom PHP shell: For remote command execution
  • Standard Linux tools: For system enumeration
