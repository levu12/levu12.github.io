OSINT Investigation: Tracking a Phishing Campaign

In this detailed OSINT investigation, I’ll walk through my methodology for tracking and analyzing a sophisticated phishing campaign that targeted financial institutions. This case study demonstrates practical OSINT techniques and tools used in real-world cybersecurity investigations.

Investigation Overview

Incident: Spear-phishing emails targeting bank employees
Timeline: December 2024 - January 2025
Scope: Multi-stage investigation across various platforms
Objective: Identify threat actors and infrastructure

Initial Evidence Collection

The investigation began with a suspicious email reported by a bank security team:

Email Analysis

Key indicators identified:

  • Suspicious domain: bankfirst-alerts.com (typosquatting)
  • Malicious link: hxxp://verification-portal[.]bankfirst-alerts[.]com/login
  • Sender IP: 185.244.25.147

Domain Investigation

WHOIS Analysis

Results revealed:

  • Registrar: Namecheap
  • Creation Date: 2024-12-20
  • Registrant: Privacy protected
  • Name Servers: Cloudflare

DNS Investigation

Using dig and passive DNS databases:

Findings:

  • A Record: 185.244.25.147
  • Historical DNS: Recently changed from 194.58.112.34
  • Subdomain enumeration: Multiple suspicious subdomains discovered

Infrastructure Mapping

IP Address Analysis

Investigating the hosting infrastructure:

Primary IP: 185.244.25.147

  • Location: Netherlands (VPS provider)
  • ASN: AS60781 (LeaseWeb)
  • Other domains: 15 additional suspicious domains hosted

Secondary IP: 194.58.112.34

  • Location: Romania
  • ASN: AS39543 (Phoenix NAP)
  • Reputation: Previously flagged for malicious activity

SSL Certificate Analysis

Certificate transparency logs revealed:

  • Issuer: Let’s Encrypt
  • Subject Alt Names: Multiple phishing domains
  • Certificate serial: Links to other campaigns

Social Media Investigation

LinkedIn Reconnaissance

Searching for the targeted organization revealed:

  • Employee profiles: 247 employees with public profiles
  • Common patterns: Job titles and department structures
  • Potential targets: IT and finance department staff

GitHub Investigation

Discovered a repository with similar phishing templates:

  • Username: sec_researcher_2024
  • Repository: email-templates (now deleted)
  • Commit history: Links to the same infrastructure

Threat Actor Profiling

Attribution Indicators

Through cross-referencing various data points:

  1. Infrastructure patterns: Consistent use of specific VPS providers
  2. Code similarities: Shared JavaScript obfuscation techniques
  3. Operational security: Timezone analysis suggests Eastern European origin
  4. Previous campaigns: Links to campaigns dating back to 2023

Tools and Techniques Used

The threat actors employed:

  • Domain generation algorithms for backup domains
  • Cloudflare for traffic filtering and protection
  • Telegram bots for credential harvesting notifications
  • URL shorteners to mask malicious links

Impact Assessment

Affected Organizations

Through pivoting on infrastructure:

  • Primary targets: 12 financial institutions
  • Secondary targets: Healthcare and government sectors
  • Geographic focus: North America and Western Europe

Campaign Scale

  • Malicious domains: 45+ identified
  • Phishing emails: Estimated 10,000+ sent
  • Success rate: Approximately 3-5% based on infrastructure logs

Mitigation and Reporting

Immediate Actions

  1. Domain blocking: All identified domains added to blocklists
  2. Threat intelligence sharing: IOCs shared with relevant organizations
  3. Law enforcement: Case referred to appropriate authorities

Long-term Monitoring

  • Infrastructure monitoring: Continued surveillance of related IPs
  • Brand monitoring: Automated alerts for similar typosquatting domains
  • Threat hunting: Proactive searching for similar campaigns

Tools and Resources Used

OSINT Tools

  • Shodan: Infrastructure reconnaissance
  • VirusTotal: Malware analysis and reputation checking
  • URLVoid: URL reputation analysis
  • Maltego: Link analysis and visualization

Custom Scripts

  • Domain monitoring: Python scripts for DNS changes
  • Social media scraping: Automated LinkedIn reconnaissance
  • Certificate transparency: SSL certificate monitoring

Databases

  • PassiveTotal: Historical DNS data
  • SecurityTrails: Domain intelligence
  • ThreatMiner: Threat intelligence aggregation

Lessons Learned

This investigation highlighted several key points:

  1. Infrastructure reuse: Threat actors often reuse infrastructure across campaigns
  2. OSINT correlation: Multiple data sources provide comprehensive attribution
  3. Timeline analysis: Understanding campaign evolution aids in prediction
  4. Collaboration: Information sharing amplifies investigation effectiveness

Conclusion

This OSINT investigation demonstrates the power of systematic intelligence gathering and analysis. By combining technical indicators with social media reconnaissance and infrastructure analysis, we successfully mapped an entire phishing operation and provided actionable intelligence for defense.

The key to successful OSINT investigations lies in methodical data collection, creative pivoting between data sources, and thorough documentation of findings.
